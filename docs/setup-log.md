## Шаг 3: Установка SSH-ключа Марка

**Время:** 09:59
**Что создано:** `~/.ssh/authorized_keys`

В этот файл был добавлен публичный SSH-ключ Марка:
```
ssh-ed25519 AAAAC3NzaC1l... mark@kreyman.com
```

**Что это значит:** SSH-ключ — это как цифровой пропуск. Существует пара: приватный ключ (у Марка на его компьютере, как личный пароль) и публичный ключ (на твоём компьютере, как замок). Когда Марк подключается, его компьютер показывает приватный ключ, а твой проверяет, что он подходит к публичному. Если совпадает — доступ открыт.

Файл `authorized_keys` — это список «кому разрешён вход». Сейчас там один ключ — Марка.

**Что это такое:**
- **Node.js** — это среда для запуска JavaScript вне браузера. Многие инструменты для разработки написаны на JavaScript и требуют Node.js для работы.
- **NVM** — менеджер версий Node.js. Позволяет устанавливать разные версии и переключаться между ними. Это нужно потому, что разные проекты могут требовать разные версии Node.js.
- **npm** — менеджер пакетов для Node.js. Как App Store, но для библиотек и инструментов программирования. Через npm устанавливаются дополнительные программы.

**Аналогия:** NVM — это как гараж с разными машинами. Node.js — конкретная машина. npm — заправка, которая позволяет добавлять «топливо» (пакеты/плагины).

**Важно:** Файл `id_ed25519` (без .pub) — это приватный ключ. Его нельзя никому отправлять, копировать, или выкладывать. Это как пин-код от банковской карты.

### Как это подключено
Марк создал **символические ссылки** (symlinks) в `~/.claude/`:
```

~/.claude/commands → ~/workspace/claude-config/commands
~/.claude/skills → ~/workspace/claude-config/skills
```

**Символическая ссылка** — это как ярлык на рабочем столе Windows. Файл `~/.claude/skills` на самом деле указывает на `~/workspace/claude-config/skills`. Claude Code ищет навыки в `~/.claude/skills/`, находит ссылку и идёт в реальную папку.

**Зачем так:** Все настройки хранятся в Git-репозитории (`claude-config`). Марк может обновить их на своём компьютере, запушить в GitHub, а потом одной командой `git pull` обновить на всех машинах где они используются.

## Шаг 10: Настройки Claude Code

**Время:** 10:23
**Что создано:** `~/.claude/settings.json`

Содержимое:
```json
{
  "skipDangerousModePermissionPrompt": true
}
```

**Что это значит:** По умолчанию Claude Code спрашивает разрешение перед каждым потенциально опасным действием (запуск команд, редактирование файлов, и т.д.). Эта настройка отключает предупреждение при запуске в «опасном режиме» (когда используется флаг `--dangerously-skip-permissions`).

**Опасный режим** означает, что Claude Code может выполнять любые действия без подтверждения пользователя. Это удобно для автоматизации, но потенциально рискованно — Claude может случайно удалить файлы или выполнить нежелательные команды.

## Шаг 11: Установка MCP Memory Keeper

**Время:** 10:41
**Что установлено:** npm-пакет `mcp-memory-keeper@0.12.1`
**Где данные:** `~/mcp-data/memory-keeper/context.db` (SQLite база, 450 КБ)

**Что это такое:**
- **MCP** (Model Context Protocol) — это стандарт для подключения внешних инструментов к Claude Code. Как USB для компьютера — позволяет подключать разные устройства.
- **Memory Keeper** — это плагин, который даёт Claude Code долговременную память. Без него Claude Code «забывает» всё после завершения диалога. С Memory Keeper он может сохранять заметки, прогресс работы и контекст проекта в базу данных.

**Как работает:** Claude Code сохраняет информацию через команды вроде `context_save(key, value)`, а в следующем диалоге может достать её через `context_get(key)`. Данные хранятся в SQLite-файле на твоём компьютере.

**Аналогия:** Представь, что у тебя есть блокнот, который Claude может использовать между разговорами. Записал что-то сегодня — прочитает завтра.

## Шаг 13: Настройка файла .zshrc

**Время:** 10:51 (финальная версия)
**Что изменено:** `~/.zshrc`

Этот файл выполняется каждый раз, когда ты открываешь терминал. В него были добавлены:

```bash
# 1. Путь к Claude Code
export PATH="$HOME/.local/bin:$PATH"

# 2. Инициализация NVM (менеджера версий Node.js)
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"

# 3. Путь к GitHub CLI
export PATH=$HOME/bin:$PATH

# 4. Алиас "yolo"
alias yolo="claude --dangerously-skip-permissions"
```

**Разбор каждой строки:**

1. **PATH** — это переменная окружения, которая говорит терминалу «в каких папках искать программы». Добавив `~/.local/bin`, терминал теперь может найти `claude`.

2. **NVM init** — загружает NVM при каждом открытии терминала, чтобы команды `node` и `npm` работали.

3. **Ещё PATH** — добавляет `~/bin` для `gh`.

4. **Алиас `yolo`** — сокращение. Вместо того чтобы каждый раз писать `claude --dangerously-skip-permissions`, можно просто набрать `yolo`. Этот режим запускает Claude Code без запроса разрешений — Claude может делать всё что угодно автоматически.

**Аналогия:** `.zshrc` — это как автозагрузка в Windows. Всё, что туда записано, выполняется при старте терминала.
