# Журнал работы: CLI Todo App

## Описание проекта

Мы с нуля создали консольное приложение для управления задачами (todo-app) на Python. Прошли полный цикл разработки: от идеи до рабочего кода, опубликованного на GitHub.

**Репозиторий:** https://github.com/FinanceAlex/cli-todo-app

---

## Шаг 1: Выбор идеи проекта

Я попросил Claude Code предложить идеи для небольшого проекта, чтобы научиться кодить с помощью Claude Code. Были предложены варианты разной сложности:

- **Для начинающих:** CLI Todo App, URL Shortener, Markdown Note Taker
- **Средний уровень:** Weather CLI, Expense Tracker, Static Site Generator
- **Продвинутый:** Chat with CSV, Personal Dashboard

Я выбрал **CLI Todo App** — консольный менеджер задач.

---

## Шаг 2: Создание репозитория и PRD

Был создан Git-репозиторий и написан **Product Requirements Document** (PRD) — документ с требованиями к продукту.

**Что описано в PRD (`docs/PRD.md`):**
- Обзор проекта и цели (обучение программированию)
- Целевой пользователь (разработчик, работающий в терминале)
- 5 основных функций: add, list, done, delete, edit
- Модель данных (id, description, status, created_at, completed_at)
- Хранение в файле `~/.todo/tasks.json`
- Интерфейс командной строки с примерами
- Нефункциональные требования (Python, без зависимостей, кроссплатформенность)
- 4 майлстоуна для поэтапной разработки

---

## Шаг 3: Написание пользовательских историй (User Stories)

Использовали специализированный навык (skill) **user-story-writer** для генерации полного набора пользовательских историй в формате JSON.

**Создано 10 историй в 4 эпиках:**

### Эпик 1 — Основа проекта (M1)

|---------|----------|------|
| US-1.1 | Инициализация хранилища при первом запуске | 1ч |
| US-1.2 | Добавление новой задачи | 1.5ч |
| US-1.3 | Отображение списка задач | 1.5ч |
| US-1.4 | Точка входа CLI и маршрутизация команд | 1ч |

### Эпик 2 — Управление задачами (M2)
| История | Название | Часы |
|---------|----------|------|
| US-2.1 | Отметить задачу как выполненную | 1ч |
| US-2.2 | Удалить задачу | 1ч |

### Эпик 3 — Редактирование и фильтрация (M3)
| История | Название | Часы |
|---------|----------|------|
| US-3.1 | Редактирование описания задачи | 1ч |
| US-3.2 | Фильтрация задач по статусу | 1ч |

### Эпик 4 — Полировка (M4)
| История | Название | Часы |
|---------|----------|------|
| US-4.1 | Команда help и сообщение об использовании | 0.5ч |
| US-4.2 | Надёжная обработка ошибок | 1.5ч |

**Каждая история содержит:**
- Описание в формате Gherkin (as a / i want to / so that)
- Обоснование (rationale) — зачем это нужно
- Критерии приёмки (acceptance criteria) — 4-6 конкретных условий
- Тестовые случаи (test cases) — 3-4 сценария с предусловиями и ожидаемыми результатами
- Технические заметки — подсказки для реализации
- Зависимости от других историй

## Шаг 4: UX-ревью всех историй

Запустили навык **review:ui-ux-review** — 4 параллельных агента (по одному на эпик) проверили все истории на предмет проблем с юзабилити.

**Найдено 64 проблемы:**
- 2 критических
- 14 высокого приоритета
- 27 среднего
- 21 низкого

**Ключевые находки:**
1. Нет обратной связи при первой инициализации хранилища
2. Нет обработки повреждённого JSON-файла
3. Пробельные описания задач принимались без ошибки
4. Нет подтверждения при удалении (деструктивная операция!)
5. Нет поддержки `--help` / `-h` (стандарт CLI)
6. Нет обработки конфликтующих флагов `--done --pending`
7. Ошибки не выводились в stderr
8. Нет подсказки "Did you mean?" при опечатках в командах
9. Нет валидации отрицательных и нулевых ID
10. Неправильное склонение ("1 tasks" вместо "1 task")

## Шаг 6: Реализация всего кода

Использовали адаптированный навык **run-epic** для автономной реализации всех историй. Код написан на Python 3, без внешних зависимостей.

### Структура проекта:
```
cli-todo-app/
├── todo.py              # Точка входа CLI (429 строк)
├── todo/
│   ├── __init__.py
│   ├── storage.py       # Модуль хранения (101 строка)
│   └── tasks.py         # Бизнес-логика (87 строк)
├── tests/
│   ├── __init__.py
│   ├── test_storage.py  # 9 тестов хранилища
│   ├── test_tasks.py    # 22 теста бизнес-логики
│   └── test_cli.py      # 49 интеграционных тестов
└── docs/
    ├── PRD.md
    └── user_stories/    # 10 JSON-файлов историй
```

### Коммиты:
1. `4c43865` — PRD и пользовательские истории
2. `afd2651` — US-1.1: Модуль хранения с атомарной записью
3. `b7e6956` — US-1.2, 2.1, 2.2, 3.1: Бизнес-логика задач
4. `8c8cb30` — US-1.3, 1.4, 3.2, 4.1, 4.2, 4.3: CLI-интерфейс
5. `a9754cc` — .gitignore

### Что реализовано:

**Модуль хранения (`todo/storage.py`):**
- Класс `Storage` с автоматической инициализацией `~/.todo/tasks.json`
- Атомарная запись через временный файл + `os.replace()`
- Иерархия исключений: `StorageError`, `CorruptedFileError`, `PermissionDeniedError`, `WriteError`
- Сообщение в stderr при первом запуске

**Бизнес-логика (`todo/tasks.py`):**
- `add()` — создание задачи с валидацией, очисткой пробелов и замиеной переносов строк
- `list_tasks()` — список с опциональной фильтрацией по статусу
- `complete()` — пометка как выполненной (идемпотентная)
- `delete()` — удаление с возвратом удалённой задачи
- `edit()` — редактирование с отслеживанием старого/нового значения и `updated_at`
- `get()` — получение одной задачи по ID

**CLI-интерфейс (`todo.py`):**
- Маршрутизация команд через словарь (dict dispatch)
- 6 команд: add, list, done, delete, edit, help
- Форматированная таблица с заголовком и обрезкой длинных описаний
- Система помощи: `--help`, `-h`, `--version`, помощь по команде (`todo help add`)
- Фильтрация: `--done`, `--pending`, `--all` с обнаружением конфликтов
- Подтверждение удаления `[y/N]` с обходом через `--force`/`-f`
- Итоговая строка с правильным склонением и контекстом фильтра
- Подсказки "Did you mean?" через `difflib.get_close_matches()`
- Все ошибки в stderr с "Error:" и подсказками Usage
- Глобальный обработчик исключений, Ctrl+C → exit 130

## Шаг 9: Демонстрация

Провели полную демонстрацию всех возможностей:

```bash
python3 todo.py add "Learn Python basics"      # Добавить задачу
python3 todo.py add "Build a CLI todo app"      # Добавить ещё
python3 todo.py list                            # Показать все
python3 todo.py done 2                          # Выполнить задачу
python3 todo.py edit 1 "Learn Python and OOP"   # Отредактировать
python3 todo.py list --pending                  # Только незавершённые
python3 todo.py list --done                     # Только завершённые
python3 todo.py delete -f 3                     # Удалить без подтверждения
python3 todo.py lst                             # Опечатка → "Did you mean: list?"
python3 todo.py --version                       # Версия → "todo v0.1.0"
```
